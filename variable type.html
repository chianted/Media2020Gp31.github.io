Java local variables


1.Local variables are declared in methods, constructors, or statement blocks;
2.Local variables are created when a method, constructor, or statement block is executed, and when they are 3.executed, the variables will be destroyed;
4.Access modifiers cannot be used for local variables;
5.Local variables are only visible in the method, constructor, or statement block in which it is declared;
6.Local variables are allocated on the stact;
7.Local variables have no default value, so after being declared, local variables must be initialized before they can be used;

Instance variable


1.Instance variables are declared in a class, but outside of methods, constructors, and statement blocks;
2.When an object is instantiated, the value of each instance variable is determined accordingly;
3.Instance variables are created when the object is created, and destroyed when the object is destroyed;
4.The value of the instance variable should be referenced by at least one method, constructor, or statement block, so that the outside can obtain instance variable information through these methods;
5.Instance variables can be declared before or after use;

Class variables (static variables)


1.Class variables are also called static variables and are declared with the static keyword in the class, but must be outside the method.
2.No matter how many objects a class creates, the class only has one copy of the class variables.
3.Static variables are rarely used except when declared as constants. Constants refer to variables declared as public / private, final, and static. Constants cannot be changed after initialization.
4.Static variables are created the first time they are accessed and destroyed at the end of the program.
